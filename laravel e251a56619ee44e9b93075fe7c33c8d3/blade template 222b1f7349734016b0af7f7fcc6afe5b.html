aikhane akta important bishoy hocce. dhorlam, amader Layout.blade.php file , menu.blade.php and footer.blade.php file 3 tai → component folder er moddhe ase. but jokhn Layout.blade.php te amra header and footer include korte jabo tokhn amader full path likhte hobe. sudhu amon likhle hobe na

@include('menu') or @include('footer')

amader full path e bole dite hobe

@include('component.menu') or @include('component.footer)

akhn sob sesh a amara Layout.blade.php ta @extend kore Home.blade.php te nilam.

but amra Layout a @yeld dea akta sction define koci. aita korbe ki child accept korbe and seta ai yeld er place a boshay dibe sequentially.

so j Layout k extend korbe ok er child component gulo @yeld er named place a pathay dite hobe tahole thik tak webpage show korbe.

so Home.blade.php jokhn Layout.blade.php k @extend korlo tokhn she tar majhe @section create korlo component name a and aitar ending o define kore dilo. tarpor korlo ki er majhe child component gulo include kore dilo. and website thik thak show korlo.

doubt clear: amra question korte pari amar Home.blade.php te @extend korlam, @include keno noy. because, @include korle @yeld kora jay na. mane component inject kora jay na.
