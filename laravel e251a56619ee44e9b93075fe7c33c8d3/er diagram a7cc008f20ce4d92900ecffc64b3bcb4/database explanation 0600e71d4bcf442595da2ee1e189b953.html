database explanation

  • user er akta profile thakteo pare abr nao pare
  • profile jodi thake then tar jonne obossoi akta user thakbe.
  • aikhane horizontal key hocce primary key r vertical key hocce foreign key
  • akta block er sathe onno r akta block er relation maintain hobe onno block er sathe j sign ase tar dara.

example: users table and profiles table. user table er sathe profile er relation kemon hobe seta bujhte profile er sathe j sign ase seta dekhte hobe. ai khetre zero or one.

abr profile er khetre user er sathe relation bujhte user er sathe j sign ase oita dekhte hobe. aikheter one and one.

mane user er profile 0 to 1, mane akta thakteo pare abr nao pare but profile er 1 to 1. mane profile er akta obossoi user thakbei thakbe.

RELATIONSHIP CONSTRAINT

this is very important. like user k parent table dhorlam and profile k child. aita the bujhano hocce, user er data gulo profile table a rakhteci. akhn jodi amon hoy j amra user k delete kore diyeci but profile roye gece. tokhn kintu inconsistency create hobe. garbage value create hobe . aitar solutin e hocce relationship constraints.
  • cascade mean akta hoile ar akta hoite hobe(parent a kono kicu update or delete hoye gele child a o seta reflect korbe)
  • restict mean agerta na hoile ami akhn amarta hoite dibo na(cascade er ulto. mane parent a kono kicu update korte chaile age child a update kore asho then parent a korba)
